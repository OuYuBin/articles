Creating Welcome page for your Product

Summary

Intro support in eclipse allows plug-ins to define pages that help introduce the product to the end user the first time the platform is started. This tutorial walks you through the process of creating a simple welcome page for an Eclipse RCP product.

 

By: Zhuo Zhang, IBM, zhuoz@cn.ibm.com

 

Introduction

Eclipse allows you to define “welcome pages” for your product. Welcome pages are intended to guide users into discovering a product's functionality. Generally, they are shown the first time a product is started.

Create an Intro part plugin

Select File > New > Project, select Plug-in Project in Plug-in Development and click Next. Input the project name as “com.sample.application.intro”, unselect Create a Java project, click Next:

 

 

Change the Plug-in Name to %pluginName and Plug-in Provider to %providerName, click Finish.

Now you can see the new created project in your package explore. In this project, create a file named “plugin.properties” in the same folder with plugin.xml.

Open plugin.properties, input following contents, and save it.

pluginName=Simple Product Intro

providerName=ZZ

Hint: for things like %variableName in plugin.xml and MAINFEST.MF, eclipse will automatically find them in plugin.properties. For example for %pluginName in above, eclipse will change it to “Simple Product Intro” when display.

Hint: it’s a good habit to define the meta-information such as name, version, and description in plugin.properties and use them by variable. It’s easy to maintain and provide international support.

In Eclipse 3.0 or above, welcome page has two kinds of modes:

Ø         Full mode (refer to home-page-id), the intro part takes over the main workbench area.

Ø         Standby mode (refer to standby-page-id), the intro part moves to the side, allowing the current perspective to remain visible.

Welcome page can switch between these two modes.

Appendix 1: source code for com.sample.application

package com.sample.application.view;

 

import org.eclipse.jface.action.Action;

import org.eclipse.ui.IViewReference;

import org.eclipse.ui.IWorkbenchPage;

import org.eclipse.ui.PlatformUI;

 

public class ShowViewAction extends Action {

 

public ShowViewAction() {

              super();

       }

       public void run() {

              IWorkbenchPage workbenchPage = PlatformUI.getWorkbench()

                            .getActiveWorkbenchWindow().getActivePage();

              IViewReference[] ivf = workbenchPage.getViewReferences();

              try {

                     int i = ivf.length;

                     for (; i-- > 0;) {

                            if (ivf[i].getId().equals(

                                          "com.sample.application.view.DefaultView")) {

                                   workbenchPage.activate(ivf[i].getView(true));

                                   break;

                            }

                     }

              } catch (Exception e1) {

                     e1.printStackTrace();

              }

       }

}
